Heterogeneous Treatment Effects

Causal calibration for treatment-effect predictions

causalCalibration calibrates treatment-effect scores after HTE estimation. It implements causal isotonic calibration, pooled cross-calibration, monotone spline calibration, overlap diagnostics, and calibration-error estimation in native Python and R interfaces.

Get started Choose a workflow Examples
  • Standard calibration and pooled cross-calibration
  • dr and r loss targets
  • Isotonic, monotone spline, linear, and histogram maps
  • Binary-treatment HTE workflows with external learners

At a glance

Interfaces fit_calibrator(), fit_cross_calibrator(), and diagnose_calibration() are available in Python and R.
Core use case Take a raw CATE score from an external learner and map it onto a calibrated effect scale with an explicit target population.
Cross-calibration Fit on pooled out-of-fold predictions, then calibrate fold-specific predictions and aggregate by the median.
Scope The current package is designed for binary-treatment HTE workflows with observed outcomes and user-supplied nuisance estimates.

Overview

What is this package for?

The package starts after HTE estimation. You bring treatment-effect predictions and the nuisance estimates required by the chosen loss. It fits a calibration map, returns calibrated scores, and estimates remaining calibration error.

Main workflow

Calibrate an existing treatment-effect model

The package wraps around the learner you already use. It only needs the prediction score and nuisance inputs. 

fit_calibrator(...)
fit_cross_calibrator(...)
diagnose_calibration(...)

Workflow helpers

Validate cross-fits and overlap before you commit

CalibrationBundle and CrossFitBundle package the inputs from an external pipeline, validate_crossfit_bundle() checks fold consistency, and assess_overlap() gives a quick package-default overlap screen before fitting.

Target population

Choose the loss before you choose the method

loss="dr" calibrates to the original study population, while loss="r" calibrates to an overlap-weighted population.

Research scope

Built for binary-treatment CATE workflows

The package currently targets binary treatment, observed outcomes, and externally estimated nuisance functions.

Diagnostics

Report calibrated models with uncertainty

Fit, compare, and report calibrated models with uncertainty rather than only applying a post-hoc map.

Calibration

Why calibration matters for CATE models

A treatment-effect score can rank units well and still miss the effect scale. Calibration fixes that scale: units with the same calibrated score should have an average treatment effect close to that score.

Step 1

Bring predictions

Start from a raw HTE score and the nuisance estimates needed to build the calibration target.

Step 2

Fit a map

Fit a monotone or parametric map from the raw score to a calibrated score.

Step 3

Predict and check

Apply the map to new scores and use diagnostics to quantify remaining calibration error.

Isotonic Calibration

Piecewise-constant monotone calibration

method="isotonic" fits a monotone LightGBM one-tree map with observation weights and a minimum leaf-size control.

Monotone Spline

Smooth monotone calibration

method="monotone_spline" estimates a smooth monotone transformation with a constrained spline fit. Use it when you want monotone calibration without a piecewise-constant map and you care about a smoother calibrated surface.

A calibrated score is not just ordered correctly; it is on the right effect scale.

Workflows

Should I use standard calibration or cross-calibration?

This choice depends on whether the upstream learner used cross-fitting and whether you saved fold-specific predictions.

Use standard calibration

Choose fit_calibrator() when you have a single prediction per observation or a separate holdout sample that you want to reserve for calibration.

Inputs: predictions, treatment, outcome, and nuisance estimates.

Use cross-calibration

Choose fit_cross_calibrator() when the learner was trained with cross-fitting and you want to calibrate in sample without a second holdout set.

Inputs: pooled OOF predictions, an n x K matrix of fold-specific predictions, and optional fold_ids.

How prediction works afterward

The implementation fits one calibration map on pooled OOF predictions, applies that map to each fold-specific column, and aggregates row-wise by the median.

Prediction contract: predict(cross_calibrator, fold_predictions)

Losses and Methods

What do we implement?

The loss chooses the target population. The method chooses the calibration map.

loss="dr"

Doubly robust calibration for the original study population. Use it when the calibrated score should target the observed population directly.

Interpretation: original-population target, with inverse-propensity weighting in the calibration target.

loss="r"

R-loss calibration for an overlap-weighted population. It is often more stable under weak overlap, but it targets a different population.

Interpretation: overlap-weighted target, usually more robust when overlap is poor.

method="isotonic"

A piecewise-constant monotone calibration map and the default starting method.

method="monotone_spline"

A smooth monotone transformation estimated by a constrained spline fit with a smoothness penalty.

method="linear"

A nonnegative-slope linear recalibration. This is the simplest low-variance choice and the easiest to interpret.

method="histogram"

Coarse piecewise-constant binning. Transparent and easy to inspect, especially as a diagnostic baseline.

Quickstart

Minimal end-to-end examples

Fit a calibrator, predict calibrated scores, then diagnose the remaining calibration error.

Python

from causal_calibration import (
    assess_overlap,
    fit_calibrator,
    diagnose_calibration,
)

overlap = assess_overlap(treatment=a, propensity=e_hat)

cal = fit_calibrator(
    predictions=tau_hat,
    treatment=a,
    outcome=y,
    mu0=mu0_hat,
    mu1=mu1_hat,
    propensity=e_hat,
    loss="dr",
    method="isotonic",
)

tau_star = cal.predict(tau_new)

diag = diagnose_calibration(
    predictions=tau_star,
    comparison_predictions=tau_hat,
    treatment=a,
    outcome=y,
    mu0=mu0_hat,
    mu1=mu1_hat,
    propensity=e_hat,
    target_population="both",
)

R

library(causalCalibration)

cal <- fit_calibrator(

  outcome = y,

tau_star <- predict(cal, tau_new)

diag <- diagnose_calibration(

  treatment = a,
  outcome = y,

  mu1 = mu1_hat,
  propensity = e_hat,
  target_population = "both"
)
Diagnostics

diagnose_calibration() estimates calibration error with balanced jackknife uncertainty, reports a BLP slope diagnostic for whether the score captures effect heterogeneity, and can report original-population and overlap-weighted targets in the same object. Use assess_overlap() before fitting when you want a quick overlap check, the default tail and ESS summaries, and the package's default loss recommendation.
